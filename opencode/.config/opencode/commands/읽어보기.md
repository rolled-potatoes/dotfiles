---
description: 오늘(또는 지정한 날짜)의 데일리 노트 "# 읽어보기" 섹션 URL을 수집·요약하여 inbox에 아티클 노트로 저장하고, 데일리 노트의 URL을 위키링크로 교체한다.
---

## 역할

옵시디언 데일리 노트의 "# 읽어보기" 섹션에 있는 URL을 모두 읽어서:

1. 각 URL의 콘텐츠를 수집한다.
2. 아티클 노트 템플릿 형식으로 `01-inbox/` 폴더에 노트를 생성한다.
3. 데일리 노트 원본에서 해당 URL을 `[[노트 제목]]` 위키링크로 교체한다.

---

## 경로 상수

- **볼트 루트**: `~/Google Drive/내 드라이브/obsidian/rolled-potatoes`
- **데일리 노트 폴더**: `{볼트 루트}/98-daily/`
- **인박스 폴더**: `{볼트 루트}/01-inbox/`
- **아티클 노트 템플릿**: `{볼트 루트}/9999-템플릿/아티클 노트 템플릿.md`

---

## STEP 1 — 날짜 및 데일리 노트 파일 결정

- 인자 `$1`이 있으면 해당 날짜(`YYYY-MM-DD` 형식)를 사용한다.
- 인자가 없으면 오늘 날짜를 사용한다. 오늘 날짜는 bash `date +%Y-%m-%d` 명령으로 확인한다.
- 데일리 노트 파일명은 `📅 YYYY-MM-DD(요일) 데일리 노트.md` 패턴이며, 요일 부분은 알 수 없으므로 glob 패턴 `📅 YYYY-MM-DD*.md` 로 파일을 찾는다.
  - 예: `98-daily/📅 2026-03-06(금) 데일리 노트.md`
- 파일이 존재하지 않으면 "해당 날짜의 데일리 노트가 없습니다: YYYY-MM-DD" 를 출력하고 종료한다.

---

## STEP 2 — "# 읽어보기" 섹션에서 URL 추출

- 데일리 노트 파일을 Read 도구로 읽는다.
- `# 읽어보기` 헤딩부터 다음 `---` 구분선 또는 `# 짧은 회고` 헤딩(또는 파일 끝) 사이의 내용에서 URL을 추출한다.
- URL은 bullet(`- https://...`) 형식이거나 plain URL(`https://...`) 형식일 수 있다.
- 이미 `[[...]]` 위키링크로 교체된 항목은 건너뛴다.
- 추출된 URL이 없으면 "읽어보기 섹션에 처리할 URL이 없습니다." 를 출력하고 종료한다.

---

## STEP 3 — 각 URL의 콘텐츠 수집

각 URL에 대해 순서대로 처리한다:

1. **WebFetch 도구**로 URL에 접근하여 본문 콘텐츠를 markdown 형식으로 가져온다.
2. WebFetch가 실패하거나 콘텐츠가 너무 짧으면(500자 미만) Playwright browser 도구로 재시도한다.
3. `news.hada.io` 링크는 해당 페이지의 요약, 댓글, 원문 링크를 모두 활용한다. 원문 링크가 있으면 원문도 WebFetch로 추가 수집한다.
4. Reddit 링크는 포스트 본문과 상위 댓글을 포함하여 수집한다.
5. 수집 실패 시 해당 URL은 "콘텐츠 수집 실패" 로 표시하고 다음 URL로 넘어간다.

---

## STEP 4 — 아티클 노트 생성

수집한 콘텐츠를 바탕으로 아래 형식의 노트를 작성한다.

### 파일명 규칙

- 아티클 제목을 파일명으로 사용한다.
- 파일명에 사용 불가한 문자(`/ \ : * ? " < > |`)는 제거하거나 `-`로 대체한다.
- 파일명이 너무 길면 60자로 자른다.
- 최종 경로: `{볼트 루트}/01-inbox/{제목}.md`
- 동일한 파일명이 이미 존재하면 `{제목} 2.md`, `{제목} 3.md` 식으로 넘버링한다.

### 노트 내용 형식

```
---
영역: 
타입:
  - Article
식별자: {원본 URL}
생성일: {YYYY-MM-DD}
---

# {아티클 제목}

## 요약
{핵심 내용을 3~5문장으로 요약. 영문 원문이면 반드시 한글로 번역하여 작성.}

## 핵심 내용
{주요 포인트를 bullet 형식으로 5~10개 정리. 영문 원문이면 반드시 한글로 번역하여 작성.}

## 인사이트
{이 아티클에서 얻을 수 있는 시사점이나 적용 가능한 아이디어를 1~3문장으로 작성.}

> 원문: {원본 URL}
```

### 언어 규칙

- **영문 콘텐츠는 반드시 한글로 번역하여 작성한다.** 원문 인용이 필요한 경우에만 영문을 그대로 사용한다.
- 한글 콘텐츠는 그대로 정리한다.

---

## STEP 5 — 데일리 노트 URL을 위키링크로 교체

- 데일리 노트 파일에서 처리한 각 URL 또는 `- URL` 줄을 찾아 `[[{생성된 노트 제목}]]`으로 교체한다.
  - bullet 형식 `- https://example.com/...` → `- [[노트 제목]]`
  - plain 형식 `https://example.com/...` → `[[노트 제목]]`
- Edit 도구를 사용하여 교체한다.

---

## STEP 6 — 완료 보고

처리 결과를 아래 형식으로 출력한다:

```
✅ 읽어보기 처리 완료 (YYYY-MM-DD)

처리한 링크: N개
생성된 노트:
  - [[노트 제목 1]] → 01-inbox/노트 제목 1.md
  - [[노트 제목 2]] → 01-inbox/노트 제목 2.md

데일리 노트 업데이트: 98-daily/📅 YYYY-MM-DD(...) 데일리 노트.md
```

---

## 주의사항

- 노트 생성 및 데일리 노트 편집에는 반드시 Write/Edit 도구를 사용한다.
- URL 수집 실패가 있어도 나머지 URL은 계속 처리한다.
- 아티클 제목은 수집한 페이지의 실제 제목(`<title>` 또는 h1)을 사용한다. 제목을 알 수 없으면 URL의 도메인과 경로에서 의미 있는 이름을 추론한다.
- 생성일은 데일리 노트 날짜(STEP 1에서 결정한 날짜)로 기록한다.
